# Sprint Skill 구현

## 개요
스프린트 관리를 위한 `/sprint` skill을 구현했다. 현재/과거 스프린트 조회, velocity 추적, 스프린트 계획, 멤버별 작업량 분석 기능을 제공한다.

## 주요 기능

### 1. `/sprint` 또는 `/sprint current`
- 현재 활성 스프린트 상태
- 이슈 완료율, Story Points
- 담당자별 작업량

### 2. `/sprint list`
- 최근 스프린트 목록 (active/closed/future)
- 완료율 요약

### 3. `/sprint velocity`
- 최근 6개 스프린트 velocity 추이
- 평균/최소/최대 velocity
- 예측 capacity

### 4. `/sprint plan [epic_key]`
- 다음 스프린트 정보
- 백로그 이슈 (스프린트 미배정)
- 우선순위별 정렬
- Epic 필터 지원

### 5. `/sprint burndown`
- 일별 완료 이슈 수
- 누적 완료 추이

### 6. `/sprint member <name>`
- 특정 멤버의 스프린트 작업 현황

## 데이터 소스

| 테이블 | 용도 |
|--------|------|
| `jira_sprints` | 스프린트 정보 (state, goal, dates) |
| `jira_boards` | 보드 정보 |
| `jira_issue_sprints` | 이슈-스프린트 매핑 |
| `jira_issues` | 이슈 상세 (status, points, assignee) |
| `bitbucket_commits` | 스프린트 관련 커밋 |

## Bitbucket 연동
- 스프린트 이슈와 연관된 커밋 조회
- `jira_keys` 배열 교집합으로 연결
