# Risk Skill 구현

## 개요
프로젝트 리스크 감지 및 관리를 위한 `/risk` skill을 구현했다. 7가지 리스크 유형을 자동으로 감지하고, 심각도를 판정하며, 해결을 추적한다.

## 리스크 유형

| Type | 설명 | 감지 기준 |
|------|------|----------|
| `delay` | 일정 지연 | 스프린트 종료 임박 + 완료율 < 50% |
| `blocker` | 블로커 | Highest/Blocker priority 미완료 |
| `velocity_drop` | Velocity 하락 | 평균 대비 20%+ 하락 |
| `dependency_block` | 의존성 차단 | 링크된 이슈 미완료 |
| `resource_conflict` | 리소스 충돌 | 8개+ 이슈 할당 |
| `scope_creep` | 스코프 증가 | 스프린트 중 이슈 추가 |
| `stale_issue` | 방치된 이슈 | In Progress 14일+ |

## 주요 기능

### 1. `/risk` 또는 `/risk summary`
- 오픈 리스크 요약 (유형별, 심각도별)

### 2. `/risk detect`
- 자동 리스크 감지 실행
- 모든 유형 스캔
- Severity 자동 판정

### 3. `/risk list`
- 리스크 목록 조회
- 유형/심각도 필터

### 4. `/risk analyze <target>`
- Epic/스프린트 리스크 분석
- 완료율, 블로커, 리소스 분석

### 5. `/risk create`
- 수동 리스크 등록

### 6. `/risk resolve <id>`
- 리스크 해결 처리

### 7. `/risk report`
- 리스크 현황 리포트 생성

## Severity 판정

| Severity | 기준 |
|----------|------|
| `critical` | 릴리즈/마일스톤 직접 영향 |
| `high` | 스프린트 목표 영향 |
| `medium` | 잠재적 영향 |
| `low` | 낮은 영향 |

## 데이터 저장

리스크는 `roadmap_risks` 테이블에 저장:
- risk_type, severity, title
- epic_key, milestone_id (연결)
- ai_suggestion (AI 대응 제안)
- status (open/resolved)
